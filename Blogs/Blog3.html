Lecture this week

This week's lecture touched base on the interactivity and interface design, it further extends on its definition of these two elements from the previous week. we spoke of the definition of interaction which is defined as a meeting point of an active body(Usually a user) and the variable it can affect and it came to be that an interface would be a symbolic representation of the meeting point that is described in the previous definition. "Interaction" was further explained and elaborated to have responsiveness that was non-random and effectively caused a change in a system internally. The concept of it as whole requires these key components, The user,The system itself (obviously), The element being manipulated, the action itself and the visual representation of the change the user caused. These combined perfectly elaborates the key elements looked for in interaction as a concept. But for it be meaningful, a visual representation is required in which an "Interface" would serve as the bridge for this interaction. In turn this would ultimately make us think how we would want our website to look and how we want the user to interact with the interface and website we tend to create. Which creates questions like "What is our target audience" ,"What are our goals" , "How do plan on mapping out our website or the structure of it" and so forth. Semantic Mark up

This a fundamental part of the Html language, this defined as the marking tags that help convey meaning to a HTML program/ a document. it is something in which helps provide meaning and structure with in our code and to give context other readers of the document. It is a universal way of expressing Html as it provides a general style guide and understanding of the document being read. it also depicted as an easier way to read a document due to its structure and organised presentation of a web page.
